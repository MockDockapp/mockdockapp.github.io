Getting Started

Core Features

1
Install & Initialize MockDock

Run the installation curl command in your terminal, then run mockdock init inside your project folder containing a docker-compose.yml file, and then visit http://localhost:11800/.

2
Activate the Universe & Enable Ghost Mode

Activate your universe by navigating to the Universe tab in the web dashboard or by running mockdock activate [project name] in your terminal. Once active, visit the Workspace screen, put your desired services in Ghost mode, and Start the Stack.

3
Configure workspace

Put services in Ghost mode and set payloads using the service control panel like enabling Stateful CRUD.

4
Apply Changes

Restart and rebuild MockDock using the Restart & Rebuild control button.

5
Save Your Configuration

Visit the Profiles tab and save the active profile for the test configuration.

6
Review Profiles Registry

Verify the newly saved profile appears in the registry list on the Profiles tab.

7
Audit Compose Configuration

Go to the Compose screen and run the Security & Health Audit on your compose file.

8
Explore Additional Settings

Click on the gear icon in the bottom-left corner of the sidebar to manage licenses or trust the Root CA cert.

Target Labs

1
Install the labs

Install the labs using the command: mockdock test-labs-install

2
Navigate to a Lab Stack

Open one of the folders inside mockdock-labs/ (e.g., payment-portal or python-advice) in your terminal.

3
Initialize & Start Universe

Run mockdock init and then visit http://localhost:11800.

4
Start the test target

Boot the test target container using Docker Compose: docker compose up -d

5
Follow the worksheets

Open the side-panel stubs, toggle Ghost Mode, and follow the step-by-step instructions inside the matching lessons/LESSON-*.md files to learn how container proxying, SSL interception, and chaos controls operate!
